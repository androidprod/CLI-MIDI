# CLI-MIDI 🎹

![License](https://img.shields.io/badge/license-MIT-blue.svg)
![C++20](https://img.shields.io/badge/C%2B%2B-20-orange.svg)
![Platform](https://img.shields.io/badge/platform-Windows%20%7C%20Linux%20%7C%20macOS-lightgrey.svg)

C++20 で実装された、超高速・軽量・インタラクティブなモダンなターミナル MIDI プレイヤー。

CLI-MIDI は、あなたのコマンドライン環境を、高い応答性を持った音楽ゲームのようなウォーターフォールUIのピアノロールに変貌させます。重いGUIフレームワークを一切必要とせず、高精度なタイミング制御、並列でのMIDIイベント送信、ゼロレイテンシーの物理ハードウェア出力を備えています。

## ✨ 特徴

- **ターミナル・ウォーターフォールUI:** ターミナルのANSI制御コードのみを駆使し、再生ラインに向かって落ちてくる「音ゲー風の未来予測（Look-ahead）」ピアノロールを完全にネイティブで描画します。
- **動的なリズムゲームビジュアル:** 再生中のテンポに応じてブロックの降下速度が自動的にスケール。30FPSで滑らかに描画されます。
- **高精度タイミング制御:** `std::chrono::high_resolution_clock` とハイブリッドスリープ（Busy-wait と OS Sleep の併用）を搭載。数千以上のノートが同時発音される激しいMIDIファイルでも、モタつき・ジッターが発生しません。
- **外部依存・設定不要:** `WinMM` (Windows)、`ALSA` (Linux)、`CoreMIDI` (macOS) を直接叩くため、OS標準機能の範囲内でコンパイル＆実行が可能です。面倒な依存ライブラリは不要です。
- **リアルタイム操作:** 再生/一時停止、±5秒のシーク、テンポの調整、音量の変更など、様々な操作をキーボードのショートカットで即座に反映します。
- **自動動的リサイズ:** ターミナルのリサイズに合わせてウォーターフォールの長さを瞬時に再計算し、表示崩れを自動で修復。

## 🚀 インストール

### バイナリのダウンロード
**Windows**, **Linux (x64)**, **Linux (ARM64)** 用のスタンドアロン・ビルド済みバイナリは [Releases](../../releases) ページから直接ダウンロードしてご利用いただけます。これらは一切のインストール作業を必要としません。

### ソースコードからビルド
ご自身でビルドを行いたい場合は、**CMake 3.20以降**、および**C++20 をサポートするコンパイラ**が必要です。

```bash
# リポジトリをクローン
git clone https://github.com/your-username/CLI-MIDI.git
cd CLI-MIDI

# Linux / macOS
./build.sh --release

# Windows (PowerShell または コマンドプロンプト)
.\build.bat
```

> **Linux における注意:** コンパイル前に `ALSA` 開発用ヘッダーがインストールされていることを確認してください。
> `sudo apt install libasound2-dev` （Ubuntu/Debianの場合）

## 🎮 使い方

実行ファイルに対して `.mid` ファイルのパスを渡すだけです。

```bash
# 標準の再生
CLI-MIDI /path/to/song.mid

# コマンドラインから 1.5倍速、音量 50% で再生を開始
CLI-MIDI -t 1.5 -v 50 /path/to/song.mid

# 接続されているすべての MIDI OUT デバイスの一覧を表示
CLI-MIDI --list-devices

# ID が 1 番の MIDI OUT デバイスを選択して再生
CLI-MIDI -o 1 /path/to/song.mid
```

### キーボードショートカット (再生中)

| キー | アクション |
|:---|:---|
| `<Space>` | 再生 / 一時停止 |
| `<` / `>` | 後ろへシーク (-5秒) / 前へシーク (+5秒) |
| `+` / `-` | テンポアップ (+0.1x) / テンポダウン (-0.1x) |
| `0` - `9` | 音量の調節 (0% ~ 90%) |
| `Q` | 終了 / 再生ストップ |

## 🛠️ アーキテクチャ構成
- **パーサー:** 様々なトラック情報を統括し、ひとつの絶対時間軸へと変換する完全な SMF (Format 0/1/2) バイナリ・パーサ。
- **シーケンサー:** 和音も一瞬で同時ディスパッチする構造と、マイクロ秒(µs)単位をハイブリッドに待機する高精度スケジューラ。
- **ターミナル UI:** 描画用のI/O処理によって音声再生スレッドがブロックされない様、アトミックなロックフリー共有により状態更新を即座に反映する30FPSループ。
